# 자리있나요 제품 기획서

## 한 줄 설명

방문자가 직접 좌석 이용 예정 시간을 업데이트하고, 다음 방문자가 매장의 잔여 좌석과 곧 비는 좌석을 확인하는 실시간 좌석 정보 플랫폼입니다.

## 핵심 사용자

- 자리를 찾는 방문자: 카페, 스터디카페, 음식점 방문 전 빈자리와 예상 대기 시간을 확인합니다.
- 매장 운영자: 초기에는 직접 입력하지 않아도 방문자 업데이트를 통해 좌석 현황이 쌓입니다.
- 광고주/매장주: 검증된 방문을 기준으로 광고비를 지불하고 매장 노출을 관리합니다.
- 플랫폼 운영자: 허위 업데이트, 포인트 악용, 신고 데이터를 관리합니다.

## MVP 핵심 기능

1. 매장 목록 확인
   - 지역, 카테고리, 매장명 검색
   - 현재 위치 기반 주변 매장 확인
   - 잔여 좌석, 곧 비는 좌석, 혼잡도 표시

2. 좌석 이용 등록
   - 사용자가 사진 기반 좌석도에서 선택한 좌석에 예상 이용 시간을 등록
   - 다른 사용자에게는 좌석 상태와 예상 종료 시간만 표시

3. 좌석도 수정
   - 사용자가 매장 사진을 올려 좌석도를 보정
   - 매장주가 좌석 추가, 삭제, 상태 변경

4. 자리 비우기와 포인트
   - 예상 종료 시간보다 일찍 비우면 포인트 지급
   - 포인트 적립은 사용자 기준 1시간에 1회만 허용
   - 포인트는 추후 쿠폰, 제휴 혜택, 랭킹, 신뢰도에 사용 가능

5. 신뢰도와 악용 방지
   - 너무 자주 등록/취소하는 사용자 제한
   - 매장 근처에서만 업데이트 가능하도록 위치 인증 추가 가능
   - 신고가 많은 정보는 가중치 낮춤
   - 매장 QR/NFC/영수증 코드로 실제 방문 여부 확인
   - 비정상 패턴은 포인트 지급 보류

## 포인트 악용 방지 설계

MVP의 localStorage 제한은 사용성 검증용입니다. 실제 서비스에서는 반드시 서버에서 제한해야 합니다.

- 서버 쿨다운: `user_point_cooldowns.last_earned_at` 기준 사용자별 1시간 1회만 지급
- 위치 검증: 매장 좌표 반경 50~100m 안에서만 이용 시작/자리 비우기 가능
- 방문 인증: 매장 테이블 QR, NFC 태그, 영수증 단기 코드 중 하나 요구
- 최소 체류 시간: 이용 시작 후 예를 들어 10분 미만 자리 비우기는 포인트 제외
- 반복 패턴 감지: 같은 매장 반복 등록, 먼 거리 매장 간 비정상 이동, 다계정 같은 기기 사용 감지
   - 매장주 정정: 매장주가 허위 좌석 상태를 신고/수정하면 해당 포인트 보류
   - 신뢰도 점수: 정확한 업데이트가 누적되면 가중치 상승, 신고/취소가 많으면 가중치 하락

## 가입과 광고 수익화

- 방문자 가입: 이메일, 전화번호 OTP, 카카오톡, 구글 로그인을 지원합니다.
- 매장주 가입: 일반 사용자 가입 후 사업자/매장 소유 인증을 거쳐 매장주 권한을 부여합니다.
- 광고 과금: 광고 노출이 아니라 검증된 방문 1건당 과금하는 방식이 자릿냥에 가장 잘 맞습니다.
- 과금 보류: 위치, QR/NFC/영수증 코드, 최소 체류 시간을 통과하지 못한 방문은 광고비를 청구하지 않습니다.
- 초기 가격 예시: 검증 방문 1건당 100~300원, 월 예산 한도 5만 원부터 시작합니다.

## 추천 기술 스택

- Frontend: Next.js 또는 React
- Styling: Tailwind CSS
- Auth: Supabase Auth
- Database: Supabase Postgres
- Realtime: Supabase Realtime
- Hosting: Vercel
- Store Search: Google Places API
- Map: Google Maps iframe 또는 Google Maps URL
- Analytics: Plausible 또는 Google Analytics
- 지도: Google Maps

## 구글 지도 연동 원칙

- 매장 검색은 Google Places API를 선택적으로 사용합니다.
- 지도 위치 확인은 API 키 없이 Google Maps iframe/URL로 먼저 제공합니다.
- 좌석 현황, 이용 시간, 포인트, 신고, 신뢰도는 자릿냥 DB에 저장합니다.
- Google API 키는 브라우저가 아니라 서버/API Route에서 우선 사용합니다.
- 검색 결과 전체를 대량 저장하지 않고, 사용자가 좌석 정보를 남긴 매장만 최소 식별 정보로 연결합니다.

## 데이터 공개 원칙

- 사용자의 이름, 연락처, 상세 방문 기록은 다른 사용자에게 보여주지 않습니다.
- 다른 사용자에게는 좌석 상태, 예상 종료 시간, 신뢰도 수준만 보여줍니다.
- 메모는 짧게 제한하고 민감 정보 입력을 유도하지 않습니다.
- 실제 서비스에서는 Row Level Security로 본인 기록만 상세 조회 가능하게 설정합니다.

## 수익화 가능성

- 프리미엄 매장 관리자 기능

- 상권 데이터 리포트

## 검증 순서

1. 특정 지역과 카테고리 하나만 선택합니다. 예: 강남 카페, 홍대 스터디카페.
2. 매장 20개 이내로 시작합니다.
3. 사용자가 좌석 정보를 자발적으로 업데이트하는지 봅니다.
4. 포인트 보상이 실제 업데이트율을 높이는지 봅니다.
5. 매장주가 이 데이터를 가치 있게 느끼는지 인터뷰합니다.
